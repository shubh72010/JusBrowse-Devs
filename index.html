JA4 Fingerprinting ‚Äî Deep Technical Breakdown
OverviewCore Architecture
‚ñº

JA4 is the modern replacement for JA3, created to fingerprint TLS clients in a way that's stable, readable, and resistant to browser randomization tricks.

It was introduced by FoxIO as part of the broader JA4+ suite.

Format

a_b_c

Example: t13d1516h2_8daaf6152771_e5627efa2ab1
SectionPurposeHuman Readable
AProtocol Metadata‚úÖ Yes
BCipher Hash‚ùå (Truncated SHA256)
CExtension Hash‚ùå (Truncated SHA256)
Part ABehavioral Header (Plaintext)
‚ñº

Example: t13d1516h2

SegmentMeaning
tTransport (TCP)
13TLS 1.3
dSNI uses Domain
15Cipher count
16Extension count
h2ALPN ‚Üí HTTP/2

This part allows quick classification without hash cracking. It separates transport, TLS version, and protocol negotiation, enabling fuzzy grouping (e.g., all TLS 1.3 TCP clients).

Part BCipher Hash
‚ñº
  • SHA256 of the sorted cipher list
  • Truncated to 12 hex characters
  • Sorting is the key upgrade

Modern browsers (especially Chrome) use GREASE and cipher randomization. JA3 would break every time cipher order changed. JA4 doesn't care ‚Äî order is normalized before hashing.

This kills cipher stunting, random order evasion, and noise inflation.

Part CExtension Hash
‚ñº
  • SHA256 of sorted TLS extensions
  • Truncated to 12 chars
  • Usually the true TLS stack signature

This is the true TLS fingerprint. OpenSSL clients cluster together, Windows SChannel separately, Go TLS stack looks different, Python's ssl module stands out.

AnalysisWhy JA3 Collapsed
‚ñº
ProblemWhy It Happened
Order-sensitiveExact sequence hashing
MD5 collisionsWeak hash
Extension randomizationBrowsers started GREASE
No metadata claritySingle opaque hash

When Chrome began shuffling TLS extension order, one browser version produced millions of JA3 hashes. Detection pipelines became unusable.

JA4 fixed it by: sorting fields, splitting metadata, using SHA256, and allowing fuzzy correlation.

SuiteJA4+ Multi-Layer Correlation
‚ñº
VariantLayerPurpose
JA4TLS ClientHelloClient fingerprint
JA4STLS ServerHelloServer fingerprint
JA4HHTTPHeader & cookie fingerprint
JA4LNetwork timingLatency analysis
JA4XX.509Certificate fingerprint

This is cross-layer fingerprinting across OSI levels. Extremely powerful when combined.

ApplicationsReal-World Security Uses
‚ñº

1. Bot & Scraper Detection

User-Agent can lie. TLS stack usually cannot.

If: UA says "Chrome 122" but JA4 matches Python requests ‚Üí Immediate red flag

2. Malware & C2 Tracking

Tools like Cobalt Strike or Sliver often ship with hardcoded TLS libraries and distinct extension patterns.

Security teams block based on JA4 hashes instead of IP. Even if attacker rotates IP, Domain, and CDN ‚Äî TLS signature persists.

3. Session Hijack Detection

If a session cookie starts with JA4 fingerprint A but suddenly switches to fingerprint B, you've likely got:

  • Cookie theft
  • Proxy injection
  • Account compromise

Immediate invalidation recommended.
